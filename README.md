## Não construo telas. Construo comportamento.

Meu trabalho em frontend não nasce da vontade de “deixar bonito”.  
Nasce da necessidade de fazer interfaces sustentarem produto de verdade.

Eu penso interface como sistema vivo:  
camada de decisão, percepção, ritmo, consistência e impacto.

Uma tela pode até parecer resolvida visualmente e ainda estar tecnicamente frágil.  
Pode parecer organizada e ainda assim esconder acoplamento, ruído, retrabalho e lentidão.  
É exatamente nesse ponto que meu trabalho começa.

Eu construo interfaces para aguentar crescimento.  
Para sustentar contexto.  
Para não desmontarem quando o produto deixa de ser apresentação e vira operação.

---

## O que me move

- arquitetura de frontend que reduz ruído e retrabalho
- interfaces orientadas a produto, não a efeito
- separação clara de responsabilidades
- sistemas de componentes com intenção
- performance como parte da experiência
- consistência visual sem sacrificar estrutura

---

## Onde meu trabalho realmente vive

Grande parte do que faço hoje está dentro de produtos reais, em ambiente privado, no ecossistema da **BitGrid**.

É lá que a parte mais importante acontece:
- evolução de fluxos
- sustentação de interface em produção
- decisões que afetam produto de verdade
- organização de frontend para contexto real, e não para vitrine

Por isso, este GitHub não existe para simular experiência.  
Ele existe para revelar **como eu penso**, **como organizo**, **como estruturo** e **como transformo complexidade em interface utilizável**.
